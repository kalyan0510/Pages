Pyramid Level Lossy Image Compression preserving Discernibility

This compression aims at reducing the file sizes as much as possible while preserving the details responsible for image subject's recognisability. 

 

Looking at both the compressed images, you can still recognise a car and a plane but they only hold minimal information of about 0.72 bits/pixel instead of their original counterparts occupying 8 bits/pixel. 

 

 

There are tons of image compression algorithms in use, like DCT, JPEG, HEIF and etcetera which can perform lossy image compressions by ignoring the high frequency components of image.  

High-frequency (HF) components correspond to “fine” details in the image while low-frequency(LF) components corresponds to “coarse” details. HF components occupy more disk space (memory)  than LF components does. 

​  = ​ + ​

 

Above equation of images delineates the basic difference between HF & LF in images. Furthermore, the details of images can be separated into images corresponding to multiple frequency bands. 

 

 DCT, JPEG & HEIF algorithms discards some of the HF/fine details in an image to reduce the image file size. Desperately compressing images to very small sizes (like less than 1 kb) with these algorithms may result in losing the recognisability of the  subject in the image. In simpler words if we compress a 1MB cat photo to 1KB photo, we might not recognise a cat in the final image. 

Here I am explaining a lossy image compression model which reduces the image disk-space/file-size but still preserving the cat details. That means,  a human or a computer who reads the compressed image must be able to recognise the cat.

 

Few things to know before we delve into the details.

Gaussian Pyramids



Pooling down an image multiple times using Gaussian filter produces multiple smaller images with exponentially decreasing sizes. All such pooled down images if put one on another forms a pyramid kinda shape. 

Laplacian Pyramid

In gaussian pyramids,  as we go up the pyramid, finer details (sharp edges or HF noises) are discarded. That means as we go down the pyramid, fined details are added from layer to layer. The concept of Laplacian pyramid is to separate out the finer details which are added to lower levels.

So each Nth Laplacian layer is the difference of upscaled N+1th and Nth Gaussian layers. 

These two are  linear multi-scale  image decomposition models. There are other pyramids which can capture the edges into a filter based on the orientation of edges. These are called  linear multi-scale, multi-orientation image decomposition models. Example: QMF Wavelet pyramids & Steerable Pyramids. More about pyramids here[https://en.wikipedia.org/wiki/Pyramid_(image_processing) ].

 

More on Laplacian Pyramid and its use in compression:

The levels of the laplacian pyramid can be compressed since each level in the pyramid has low variance and entropy and each level image may be represented at very low sample density.

Laplacian levels have low variance because, each level mostly stores only the edges, is filled with near zero values. Only few number of pixels lying on the edges are away from zero. This means we don’t need 255 values (8 bits) to represent a pixel in each colour channel. Thus we can save some bits by decreasing the color levels. This process is called quantisation.

 

Convolutional Neural Networks:

CNNs are the most famous algorithms among biologically inspired AI techniques. Deep CNN are proved to be very similar to Human visual cortex, which means  we can make a computer recognise an image the same way we Humans recognise objects with our eyes. Thus we can evaluate the recognisability of an object on a linear scale. 

In this post, I will be using the CNN for the same purpose: To evaluate the recognisability of a subject(cat or dog / car or plane) from compressed image. 

 

Lets see the model:

Now we know one way of decomposing the image and one way of evaluating the image’s recognisability. Unlike JPEG compression which is loosely based on human psycho-visual system, we will direct the image compression directly with the CNNs feedback on image recognisability. 

 

First we decompose the image into multiple levels of laplacian pyramids. Now we check the proportions of all layers contributing to image recognisability. The layer which is contributing the least will be compressed (lossily) more.

Now again we will check the contributions of each layer to the image’s recognisability and re adjust the proportions. This will be repeated until the CNN shows ambiguity in recognition. 

Finally, we will compress all the pyramid layers(lossily) in the optimal proportions.

 

 

 

 

Resources:

A set of images of either a car or a plane

A CNN for classifying these images. The CNN outputs the confidence values of each class.

 

 

Algorithm

  1. Read an image

            im = imread(“car.jpg")

1. Construct pyramid from the Image:

Given an Image, I decompose the details of Image corresponding to multiple frequencies into multiple levels [p0,p1, .. pL] of a pyramid. Using any of the image decomposition algorithms, like, Laplacian Pyramid model, QMF/Wavelet Transform pyramids or etc.

Here I am using Laplacian Pyramids

laplacianPyr = genLapPyr(im)

(Each level of pyramid shown in a clockwise spiral way)

 

 

1.     Initialise Quality vector Q = [q0,q1, .. ,qL] = [1.0, 1.0, .. , 1.0]

            Q vector is maintained to keep track of compression ratio of each level in the pyramid.

            Q = [0.5, 1.0, 1.0, 0.2, 1.0] means, we compress level 0 of pyramid by factor of 2 and level 3 by factor of 5.  This is a lossy compression which takes quality value and compresses the image to that. (For this implementation I used pixel value quantisation and cv2’s resizing with INTER_NEAREST interpolation)  

            The goal of this algorithm is to find the minimum values for Q while maintaining the recognisability of image reconstructed from compressed pyramids.

 

         Q = [1,1, .. , 1]

 

 

1.     Now we need to decrease the Q vector values. A level which is contributing least for the image recognisability can be compressed more. It makes sense since we are discarding the useless information. 

            For example: To find the contribution of a  level X, we will drop some details in it and reconstruct the image (ImX). This image will be fed to CNN for finding the confidence value of the image subject’s class. In the example, we will find the CNN’s confidence of the image being a car. 

            Basically we discard some details in the level and see how recognisable the reconstructed image is.

            If the CNN’s confidence is relatively good after dropping details in a level, the level has less contribution. 

            This way we get the contributions (w0, w1, .., wL) of all the levels and adjust the Q vector accordingly.

 

           

            Q_i = [ q0,q1,.. qi/2, .. , qL ]  // discard more details in its level

            compressedPyr = compressPyr( laplacianPyr, Q_i) // compress lapPyr according to Q values

 

            wi =  convNN_confidence( reconstructIm(compressedPyr)  )    // find contributions

           

            Q = Q - epsilon * [w0, w1, .., wL]

           

 

 

4. We repeat the above process until the confidence of the reconstructed image from compressed pyramids is less than a threshold.

 

6. Using the vector we compress all the layers of the pyramid and save the pyramids. When ever we want to open the photo, the image can be reconstructed form the compressed pyramid.

 

 

Other ways to improve this type of compression:

  1. We can use QMF / Wavelet decomposition technique for generating the pyramids. But this can be more complicated yet robust as we have 3 decompositions in each layer.

 

 A QMF decomposition of our sample image

  1. Instead of decomposing the image and discarding details in each layer, we can directly discard some segments in the image by replacing with zero. In other words, we segment the image into multiple areas and drop some of them while maintaining the subjects recognisability. Ideally this should crop the subject out of its background.

 

 

Story behind:

Earlier I had worked on Radio Frequency Data to mitigate the affect of interference while classifying a RF signal as Pulsar star emitted signal.  For that, I generated a CNN for classification of interferences and pulsar emissions. I wanted to compress the RF data but without losing the RF signals.  This compression algorithm had shown more impressible results on day-to-day life photos than on RF data 😅.
